# 账单系统接口文档

## 1. 基本约定

- 基础地址：`http://localhost:8080`
- 在线文档：后端启动后访问 `/swagger-ui/index.html` 或 `/v3/api-docs`。本文档基于 Controller、DTO、Service 与前端实际调用补充业务说明。
- 鉴权方式：除 `/api/auth/**`、Swagger、WebSocket 握手外，接口均需请求头 `Authorization: Bearer <accessToken>`。
- 请求格式：JSON 接口使用 `Content-Type: application/json`；文件上传使用 `multipart/form-data`。
- 统一响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- 导出接口和文件内容接口直接返回二进制流，不包裹统一响应。
- 参数标记：`必填` 表示后端校验或方法签名要求；`可选` 表示可省略或有默认值。
- 通用实体字段：实体返回通常包含 `id`、`createdTime`、`updatedTime`、`createdBy`、`updatedBy`、`deleted` 等基础字段，示例仅展示核心业务字段。

## 2. 错误码定义

| code | message | 含义 | 常见场景 |
|---:|---|---|---|
| 200 | success | 请求成功 | 正常返回 |
| 400 | bad request / 具体校验信息 | 请求参数错误 | 参数缺失、JSON 格式错误、字段校验失败、上传文件过大 |
| 401 | unauthorized / 登录已失效 | 未认证或令牌失效 | 未携带 Token、Token 未在 Redis、刷新令牌无效 |
| 403 | forbidden | 无权限访问 | 管理端接口缺少对应权限编码 |
| 404 | not found | 资源不存在 | 预留错误码 |
| 405 | 请求方法不被允许 | HTTP 方法错误 | 使用了未支持的 PUT/PATCH 等 |
| 415 | 请求内容类型不被支持 | Content-Type 错误 | 上传/JSON 类型不匹配 |
| 429 | too many requests | 请求过于频繁 | 登录限流等场景 |
| 460 | 业务错误信息 | 业务规则错误 | 用户名已存在、账本/账户/分类不存在、无权访问文件 |
| 500 | server error | 服务端异常 | 未处理异常 |

错误响应示例：

```json
{
  "code": 460,
  "message": "账本不存在",
  "data": null
}
```

## 3. 用户端接口

### 3.1 认证接口

#### POST `/api/auth/login` 用户登录

- 鉴权：无需
- Content-Type：`application/json`

请求参数：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| username | Body | string | 是 | 用户名 |
| password | Body | string | 是 | 密码 |

请求示例：

```json
{
  "username": "demo",
  "password": "Demo123456"
}
```

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": 2,
    "username": "demo",
    "nickname": "演示用户",
    "accessToken": "eyJhbGciOiJIUzI1NiJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiJ9...",
    "roles": ["ROLE_USER"],
    "permissions": ["bill:list", "bill:create"]
  }
}
```

业务说明：用户状态为禁用时无法登录；登录成功后 Token 写入 Redis，访问接口时必须携带 accessToken。

#### POST `/api/auth/register` 用户注册

- 鉴权：无需
- Content-Type：`application/json`

请求参数：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| username | Body | string | 是 | 用户名，需唯一 |
| nickname | Body | string | 是 | 昵称 |
| password | Body | string | 是 | 8-20 位，且同时包含字母和数字 |
| confirmPassword | Body | string | 是 | 确认密码，必须与 password 一致 |
| email | Body | string | 否 | 邮箱，需符合邮箱格式 |

请求示例：

```json
{
  "username": "newuser",
  "nickname": "新用户",
  "password": "Newuser123",
  "confirmPassword": "Newuser123",
  "email": "newuser@example.com"
}
```

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": 3,
    "username": "newuser",
    "nickname": "新用户",
    "accessToken": "eyJ...",
    "refreshToken": "eyJ...",
    "roles": ["ROLE_USER"],
    "permissions": []
  }
}
```

业务说明：注册成功后会创建用户、绑定普通用户角色、创建默认账本并返回登录态；用户名重复或两次密码不一致返回 `460`。

#### POST `/api/auth/refresh` 刷新令牌

- 鉴权：无需
- Content-Type：`application/json`

请求参数：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| refreshToken | Body | string | 是 | 刷新令牌 |

请求示例：

```json
{
  "refreshToken": "eyJhbGciOiJIUzI1NiJ9..."
}
```

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": 2,
    "username": "demo",
    "nickname": "演示用户",
    "accessToken": "new-access-token",
    "refreshToken": "new-refresh-token",
    "roles": ["ROLE_USER"],
    "permissions": ["bill:list"]
  }
}
```

#### POST `/api/auth/logout` 退出登录

- 鉴权：可选；建议携带当前 accessToken

请求参数：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| Authorization | Header | string | 否 | `Bearer <accessToken>`，携带时会清理 Redis 中的 Token |

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

### 3.3 账本接口

#### GET `/api/books` 查询当前用户账本

- 鉴权：Bearer Token

请求参数：无

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "userId": 2,
      "bookName": "默认账本",
      "currencyCode": "CNY",

    }
  ]

```

#### POST `/api/books` 新增或更新账本

- 鉴权：Bearer Token
- Content-Type：`application/json`

请求参数：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| id | Body | long | 否 | 账本 ID，传入表示更新 |
| bookName | Body | string | 是 | 账本名称 |
| currencyCode | Body | string | 否 | 币种，默认 `CNY` |
| isDefault | Body | integer | 否 | 是否默认账本：`1` 是，`0` 否，默认 `0` |

请求示例：

```json
{
  "bookName": "家庭账本",
  "currencyCode": "CNY",
  "isDefault": 1
}
```

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 5,
    "userId": 2,
    "bookName": "家庭账本",
    "currencyCode": "CNY",
    "isDefault": 1
  }
}
```

业务说明：设置为默认账本时，会将当前用户其他账本的 `isDefault` 更新为 `0`。

#### DELETE `/api/books/{bookId}` 删除账本

请求参数：

| 参数 | 位置 | 类型 | 必填 | 说明 |

## 5. 枚举与业务取值

| 字段 | 取值 | 说明 |
|---|---|---|
| billType / categoryType | `EXPENSE` | 支出 |
| billType / categoryType | `INCOME` | 收入 |
| sourceType | `MANUAL` | 手动记账；空值也会归一为手动记账 |
| sourceType | `IMPORT` | Excel/微信账单导入；兼容 `EXCEL`、`Excel导入` |
| sourceType | `OCR` | OCR 导入；兼容 `OCR识别`、`OCR导入` |
| sourceType | `AUTO` | 自动记账；兼容中文 `自动记账` |
| publishStatus | `1` / `0` | 发布 / 草稿或下线 |
| status | `1` / `0` | 启用 / 禁用 |
| readStatus | `1` / `0` | 已读 / 未读 |
| isDefault | `1` / `0` | 默认账本 / 非默认账本 |
